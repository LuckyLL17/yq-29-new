# 纸浆模塑模具设计助手 — 数据流追踪与核心模块依赖图

## 1. 项目概览

本项目是一个基于 React + Three.js 的 **纸浆模塑模具设计辅助工具**，提供 3D 模型可视化与模具可制造性分析能力。

| 维度 | 技术选型 |
|------|---------|
| 前端框架 | React 18 + TypeScript |
| 3D 渲染 | Three.js + @react-three/fiber + @react-three/drei |
| 状态管理 | Zustand 5（单一全局 Store） |
| 路由 | React Router 7 |
| 图表 | Recharts |
| 样式 | Tailwind CSS 3 |
| 构建工具 | Vite 6 |

### 3.2 状态中心依赖图（Store → Utils）

```
                    ┌─────────────────────┐

## 4. 关键数据流追踪

### 4.1 模型加载流

```
用户点击"导入模型1"
    │
    ▼
LeftToolbar.handleFileUpload()
    │  setIsLoading(true)
    │  loadModelFromFile(file)
    │      ├── FileReader 读取文件
    │      ├── STLLoader / OBJLoader 解析几何体
    │      ├── geometry.center() + computeVertexNormals()
    │      └── geometryToModelData() → ModelData
    │            ├── 提取 vertices / indices
    │            ├── computeFaceNormals()
    │            └── computeBoundingBox()
    │
    ▼
store.setModel(modelData, fileName)
    │  同时重置所有分析结果: draftAngleResult=null, wallThicknessResult=null,
    │  drainHoleResult=null, cycleResult=null, sectionResult=null, modelDiffResult=null
    │
    ▼
ModelViewer 订阅 model → 触发 Scene 重渲染
    │  displayModel = model || createSampleBoxModel()
    │  Scene → ModelMesh 渲染新网格
    │
    ▼
setIsLoading(false)
```

### 4.2 脱模角度分析流

```
用户点击"脱模角度"工具
    │
    ▼
LeftToolbar.runAnalysis('draft')
    │  setIsLoading(true)
    │  setAnalysisMode('draft')
    │
    ├── analyzeDraftAngles(model, draftDirection, draftAngleThreshold)
    │      │
    │      ├── computeFaceNormals(vertices, indices)
    │      │     → Float32Array (每面3分量法线)
    │      │
    │      ├── 遍历每面: dot(faceNormal, draftDir) → 角度
    │      │     faceAngles[i] = 90 - acos(dot)  (脱模角)
    │      │
    │      ├── 识别倒扣面 (angle < threshold)
    │      │
    │      ├── findConnectedUndercutRegions()
    │      │     → BFS 连通域搜索
    │      │     → 每区域计算: centroid, area, dimensions, severity
    │      │
    │      ├── generateRepairSuggestions()
    │      │     → 根据 severity/area/dimensions 生成修复建议
    │      │
    │      └── computeAngleDistribution()
    │            → 按阈值分段统计
    │
    ▼
store.setDraftAngleResult(result)
    │  selectedUndercutRegionId = null (重置选中)
    │
    ▼
[ModelMesh] 读取 analysisMode='draft' + draftAngleResult
    │  ├── highlightUndercuts=true → 基体半透明 + UndercutRegionsDisplay 3D高亮
    │  └── highlightUndercuts=false → createDraftAngleVertexColors() 热力图着色
    │
    ▼
[RightPanel] 读取 analysisMode='draft'
    │  展示: 角度统计卡片 + 倒扣区域列表 + 修复建议 + DraftAngleChart
    │
    ▼
setIsLoading(false)
```

### 4.3 壁厚分析流

```
用户点击"壁厚分析"工具
    │
    ▼
LeftToolbar.runAnalysis('thickness')
    │
    ├── analyzeWallThickness(model, sampleCount)
    │      │
    │      ├── buildBVH(vertices, indices) → 三角面列表
    │      ├── computeFaceNormals()
    │      │
    │      ├── 对采样面发射反向法线射线
    │      │     raycastThickness(origin, dir, bvh.triangles)
    │      │       → rayTriangleIntersect() 遍历求最近交点
    │      │       → thickness = 射线距离 t
    │      │
    │      └── computeThicknessDistribution() → 8段直方图
    │
    ▼
store.setWallThicknessResult(result)
    │
    ▼
[ModelMesh] 读取 analysisMode='thickness'
    │  ├── showThicknessHeatmap=true → createThicknessVertexColors() 顶点着色
    │  └── showThicknessHeatmap=false → 半透明蓝 + ThicknessSamplesDisplay 球体
    │
    ▼
[RightPanel] 读取 analysisMode='thickness'
    │  展示: 最薄/平均/最厚 + 定位按钮 + 配色方案 + ThicknessChart
    │
    ▼
store.focusOnThinnestPoint() / focusOnThickestPoint()
    │  → 设置 cameraFocusTarget → Scene 中 OrbitControls 平滑动画聚焦
```

### 4.4 滤水孔规划流

```
用户点击"滤水孔"工具
    │
    ▼
LeftToolbar.runAnalysis('holes')
    │
    ├── planDrainHoles(model, diameter, spacing, depth)
    │      │
    │      ├── 遍历所有面 → 计算优先级 (高度权重 + 法线权重)
    │      ├── 按 priority 降序排列
    │      │
    │      ├── 贪心放置: 检查与已有孔的间距 ≥ holeSpacing
    │      │     高优先级 → suction (吸水孔), 直径×1.2
    │      │     低优先级 → dewatering (脱水孔)
    │      │
    │      └── estimateSurfaceArea() → 计算开孔密度
    │
    ▼
store.setDrainHoleResult(result)
    │
    ▼
[ModelViewer/Scene] analysisMode='holes'
    │  → DrainHolesDisplay 渲染圆柱体表示孔位
    │  → DrainHoleInteraction 处理 3D 交互 (添加/删除/移动)
    │
    ▼
[RightPanel] 展示孔列表 + 参数编辑 + 导出
    │  ┌── 手动编辑模式: addDrainHole / updateDrainHole / removeDrainHole
    │  │     → 碰撞检测: checkHoleCollision() → autoAvoidCollision()
    │  │
    │  └── 阵列生成: generateRectangleArray / generateCircleArray
    │        → 生成后碰撞过滤
    │
    ▼
导出流程:
    │  exportToDXF(holes)   → 下载 .dxf 文件
    │  exportToCoordinates() → 下载 .csv 坐标列表
```

### 4.5 截面分析流

```
用户点击"截面分析"工具
    │
    ▼
LeftToolbar.runAnalysis('section')
    │
    ├── getPlaneBounds(model, axis) → 截面范围
    ├── setSectionPlane({ visible, position, axis })
    │
    ├── computeSection(model, plane, resolution)
    │      │
    │      ├── 逐面计算与截面平面交点 → 生成线段段(Segment)
    │      ├── buildContoursFromSegments() → BFS 连接线段形成轮廓
    │      ├── computeArea() / computePerimeter()
    │      │
    │      └── computeSectionThickness()
    │            → 沿轮廓等距采样 → raycastThickness() 射线测壁厚
    │
    ▼
store.setSectionResult(result)
    │
    ▼
[ModelViewer/Scene]
    │  ├── SceneClippingSetup → gl.clippingPlanes 设置 Three.js 裁剪面
    │  ├── SectionPlane → 可视化截面平面 (半透明矩形)
    │  └── SectionContour → 绘制截面轮廓线 + 壁厚采样点
    │
    ▼
[RightPanel] 展示截面积/周长 + SectionThicknessCurve
    │
    ▼
截面位置滑块变更时:
    │  store.setSectionPosition(pos)
    │  → Scene useEffect 监听 sectionPlane.position → computeSection() → setSectionResult()
    │  → SceneClippingSetup 同步更新 gl.clippingPlanes
```

### 4.6 模型对比流

```
用户点击"模型对比"工具 (需先加载 model + model2)
    │
    ▼
LeftToolbar.runAnalysis('compare')
    │  setAnalysisMode('compare')
    │
    ├── compareMode='diffcolormap' → computeModelDiff(model, model2)
    │      │
    │      ├── 构建模型2 BVH
    │      ├── 逐顶点: pointToMeshDistance() → 最近点距离
    │      ├── computeSignedDistance() → 射线法判定正负(凸出/凹陷)
    │      └── computeDistanceDistribution()
    │
    │  compareMode='overlay'/'sidebyside' → 无需计算, 仅视觉对比
    │
    ▼
store.setModelDiffResult(result) / null
    │
    ▼
[ModelViewer] analysisMode='compare' → CompareScene
    │  ├── overlay → 两模型叠加, 透明度控制
    │  ├── sidebyside → offsetX 偏移并排显示
    │  └── diffcolormap → DiffModelMesh (顶点着色: 红=凸出, 蓝=凹陷)
    │
    ▼
DiffColorLegend / 对比参数面板
```

### 4.7 成型周期预估流

```
用户点击"成型周期"工具

                       ──→ SceneClippingSetup → gl.clippingPlanes

layersEnabled ──→ Scene → ModelMesh vs LayeredModelMesh 切换

## 9. 依赖关系总结

### 9.1 模块耦合度矩阵

| 模块 | geometry | types | store | Three.js | Zustand |
|------|----------|-------|-------|----------|---------|
| geometry.ts | - | ✅ | - | - | - |
| modelLoader.ts | ✅ | ✅ | - | ✅ | - |
| draftAngle.ts | ✅ | ✅ | - | - | - |
| wallThickness.ts | ✅ | ✅ | - | - | - |
| drainHoles.ts | ✅ | ✅ | - | - | - |
| section.ts | ✅ | ✅ | - | - | - |
| modelDiff.ts | - | ✅ | - | ✅ | - |
| moldingCycle.ts | - | ✅ | - | - | - |
| layerSplitter.ts | - | ✅ | - | ✅ | - |
| useAppStore.ts | - | ✅ | - | - | ✅ |
| LeftToolbar | - | ✅ | ✅ | - | ✅ |
| ModelViewer | - | - | ✅ | ✅ | - |
| ModelMesh | - | - | ✅ | ✅ | - |
| RightPanel | - | ✅ | ✅ | - | ✅ |

### 9.2 关键观察

1. **geometry.ts 是基础运算层**：几乎所有分析模块都依赖它，它仅依赖 types，零框架依赖
2. **Store 是唯一双向依赖点**：Store 的 drainHole 相关 action 反向调用了 drainHoles.ts 的碰撞检测函数
3. **LeftToolbar 是调度枢纽**：所有分析计算的触发都经由 LeftToolbar.runAnalysis()
4. **ModelViewer 是渲染枢纽**：根据 analysisMode 切换 Scene/CompareScene，消费所有分析结果
5. **RightPanel 是展示枢纽**：纯只读消费 Store 中的分析结果，无直接 utils 调用
6. **moldingCycle.ts 完全独立**：纯经验公式，无几何依赖
7. **modelDiff.ts 独立于 geometry.ts**：直接使用 Three.js 的 BVH 和 Raycaster
